# e-Library Management System
# Version History

## Version 1.0.0 - November 26, 2025

### Initial Release - Full Feature Implementation

#### Core Features Implemented

**Circulation System:**
- ✅ Check-out/Check-in with Item ID or ISBN (barcode scanning optional)
- ✅ Automated due date calculation
- ✅ Item renewals (up to 2 times)
- ✅ Item status tracking (8 statuses)
- ✅ In-transit management between locations
- ✅ Borrower registration and management
- ✅ Borrowing limits enforcement
- ✅ Borrower blocking/suspension with reasons
- ✅ Hold queue management with positions
- ✅ Hold shelf management
- ✅ Automated email notifications (overdue, pre-due, hold ready)
- ✅ Statistical reports (overdue, circulation stats)

**Public Access Catalog (OPAC):**
- ✅ Keyword search across all fields
- ✅ Field-specific search (Title, Author, Subject, Call Number, ISBN)
- ✅ Advanced search with Boolean capability
- ✅ Faceted search filters (Type, Language, Date, Availability)
- ✅ Publication browsing by type, subject, and author
- ✅ Real-time item status display
- ✅ Due date display for checked-out items
- ✅ Hold queue position visibility
- ✅ Personal account management
- ✅ Online renewal capability
- ✅ Hold placement and cancellation
- ✅ Borrowing history view

**Publication Types:**
- ✅ Manuals (MAN)
- ✅ SOPs - Standard Operating Procedures (SOP)
- ✅ Capstone Projects (CAP)
- ✅ TTPs - Tactics, Techniques, and Procedures (TTP)

#### Technical Implementation

**Backend:**
- Django 4.2 framework
- Custom User model with library features
- Comprehensive database models
- Celery task queue for notifications
- Redis for Celery broker
- Class-based and function-based views
- Django forms with validation
- Django admin customization

**Frontend:**
- Bootstrap 5 responsive design
- Bootstrap Icons
- django-crispy-forms for form rendering
- Custom CSS and JavaScript
- Mobile-responsive templates
- Print-friendly report layouts

**Database Schema:**
- User (Custom with library fields)
- Publication (bibliographic records)
- Item (physical/digital copies)
- PublicationType
- Author
- Subject
- Publisher
- Location
- Loan (checkout records)
- Hold (reserve requests)
- InTransit (item transfers)
- Notification (email tracking)

#### Configuration Options

All configurable in settings.py:
- LIBRARY_NAME
- MAX_ITEMS_PER_BORROWER (default: 5)
- LOAN_PERIOD_DAYS (default: 14)
- RENEWAL_LIMIT (default: 2)
- PRE_DUE_NOTICE_DAYS (default: 2)
- OVERDUE_GRACE_PERIOD_DAYS (default: 7)

#### Security Features

- CSRF protection
- Password validation
- User authentication/authorization
- Role-based access control
- Session management
- SQL injection prevention
- XSS protection

#### Known Limitations

- Email requires Celery + Redis setup
- SQLite suitable for development only
- Single-language interface (English)
- No fine/fee calculation
- No barcode generation
- No patron import/export
- No Z39.50 integration

#### System Requirements

**Minimum:**
- Python 3.8+
- 100MB disk space
- 512MB RAM

**Recommended:**
- Python 3.10+
- 1GB disk space
- 2GB RAM
- Redis server (for notifications)

**Supported Platforms:**
- Windows 10/11
- Linux (Ubuntu, Debian, etc.)
- macOS 10.15+
